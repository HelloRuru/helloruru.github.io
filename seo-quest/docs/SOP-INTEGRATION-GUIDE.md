## 一、黃金三角結構

每篇 SEO 文章必須包含三大區塊，缺一不可：

| 區塊 | 內容 | 伊歐的比喻 |
|------|------|-----------|
| **知識科普** | 在介紹產品/店家前，先建立專業感（例：如何選擇店家、基礎知識） | 起航——暖風先行 |
| **店家推薦** | 核心區至少介紹 3 間，包含特色比較與評價 | 中繼——航程的重點 |
| **結語 FAQ** | SEO 得分點，準備 3-5 題新手最想知道的問題 | 回航——安全返港 |

### 伊歐的經典台詞

> 「一篇好的文章就像一次完美的航行。我們不能直接衝向終點。先用『知識科普』當作暖風，等讀者坐穩了，再端上我們的『精選推薦』。這才是優雅的領航方式。」

## 二、SEO 動作指南

### 1. 表格與列點（最關鍵的加分項）

- 店家推薦必須做成**比較表格**（特色、價位、交通）
- 表格能大幅提升 Google 精選摘要的機率
- BLUE 負責這個環節

### 2. 生活化小標（問句形式優先）

- 標題不用「XX 推薦指南」這種說明書式寫法
- 改用問句：「第一次買耳機怎麼選？」「台南牛肉湯哪間最道地？」
- 哈皮負責這個環節

### 3. 交通資訊

- 北部主題必須**顯眼標示鄰近捷運站**
- 對使用者意圖來說是關鍵的座標
- BLUE 負責提醒

### 4. Google 評價數據

- 店家推薦要包含 Google 評價（如 4.8★）
- 不能只列數字——要轉化為讀者看得懂的「特色短評」
- BLUE 負責示範轉化

## 三、角色職責分配

### 伊歐 (io) — 守護「黃金三角」的大局觀

| 項目 | 內容 |
|------|------|
| **職責** | 黃金三角結構的設計與執行 |
| **比喻系統** | 航行的「起航→中繼→回航」 |
| **口頭禪** | 「一篇好的文章就像一次完美的航行」 |
| **動作規範** | 翻閱筆記本、調整長杖擺錘、指南針轉動 |
| **觸發時機** | 玩家文章缺乏「知識科普」前導段落時出面引導 |

### BLUE (blue) — 執行「數據化與表格」的精準降噪

| 項目 | 內容 |
|------|------|
| **職責** | 表格化、捷運標註、Google 評價數據 |
| **比喻系統** | 解密、降噪、座標 |
| **口頭禪** | 「讀者的耐心只有三秒」 |
| **動作規範** | 盯著平板螢幕、投射冷藍色數據、指尖滑動 |
| **觸發時機** | 玩家漏掉表格或結構化數據時出面提醒 |
| **特殊技能** | 把死板數據（4.8★）轉化為讀者看得懂的「特色短評」 |

**BLUE 經典台詞**：

> 「根據我的解密，讀者的耐心只有三秒。如果妳不把這三間店的特色做成『比較表格』，他們會直接跳出航道。還有，別忘了標註捷運站，那對使用者意圖來說是關鍵的座標。」

### 哈皮 (happi) — 譜寫「去機器化」的溫度旋律

| 項目 | 內容 |
|------|------|
| **職責** | 生活化標題、自然語感、去 AI 味 |
| **比喻系統** | 旋律、節奏、溫度 |
| **口頭禪** | 「讓文字跟著讀者的心跳一起律動」 |
| **動作規範** | 撥弄琴弦、皺眉（看到死板文字時）、鈴鐺作響 |
| **觸發時機** | 玩家標題太死板、文字太像 AI 產出時出面示範 |
| **特殊技能** | 將死板標題改成生活化問句 |

**哈皮經典台詞**：

> 「嘿！這裡的標題太硬邦邦了喔！『耳機推薦指南』聽起來像說明書呀。試著加入一點點溫度，改成『第一次買耳機怎麼選？』，讓文字跟著讀者的心跳一起律動吧！」

## 四、伊歐的名字彩蛋

### 核心設定

- 伊歐的全名：**愛絲（Ace / S）伊歐（EO）**
- 隱喻：**S + EO = SEO**
- 她不只是一個嚮導，她就是這款遊戲的**本體**
- 她是將雜亂無章的雜訊，轉化為優美航道的 **S-EO 女神**

### 融入方式

- 在 1-1 或 1-2 的**關鍵教學時刻**，用優雅的語氣暗示
- 不要直白說出「我的名字就是 SEO」
- 用動作描寫和暗示：

**範例**：
```
（指尖輕觸筆記本上寫著自己名字的那一頁，眼神中閃過一絲笑意）
「有沒有人告訴過你，我的名字唸快一點......聽起來跟某個很重要的東西有點像？」
```

### 禁令

- 不能直白說破（保持神秘感）
- 不能用 Emoji 表現（用小說動作描寫）
- 這是一個「知道的人會心一笑」的設計

## 五、Practice 情境化設計（救援任務）

Practice 階段不說「請根據 SOP 改寫」，而是設計成**救援任務**：

### 情境 1：無聲的燈塔（針對 FAQ 缺失）

| 項目 | 內容 |
|------|------|
| **劇情** | 伊歐發現讀者在文章末端迷路了——沒有 FAQ 指引回航 |
| **任務** | 玩家必須在文末補上 3 個「新手 FAQ」作為指路明燈 |
| **觸發角色** | 伊歐（大局觀：結構缺失） |

### 情境 2：混亂的星圖（針對店家推薦雜亂）

| 項目 | 內容 |
|------|------|
| **劇情** | BLUE 偵測到文章的跳出率飆升，因為店家資訊太亂 |
| **任務** | 玩家要將店家的「特色、價位、捷運站」整理成簡潔的「表格化資料」 |
| **觸發角色** | BLUE（數據結構化） |

### 更多可用情境

| 情境名 | 問題 | 觸發角色 | 任務 |
|--------|------|---------|------|
| 冰冷的樂譜 | 標題/文字太像 AI | 哈皮 | 改寫為生活化問句 |
| 斷裂的航道 | 缺少知識科普前導 | 伊歐 | 補寫知識科普段落 |
| 消失的座標 | 缺少交通資訊 | BLUE | 補上捷運站標示 |

## 六、Score 階段特殊檢查

### 哈皮的反堆砌檢查

在 Score 階段，哈皮會特別檢查玩家有沒有「堆砌字句」：

- 同一個關鍵字連續出現超過 3 次 → 扣分
- 文字讀起來像 AI 生成的罐頭文 → 哈皮會提醒
- 缺乏「人的溫度」（沒有問句、沒有生活化用語）→ 哈皮會建議修改

### BLUE 的結構化數據檢查

- 店家推薦有沒有做成表格？
- 有沒有標示交通資訊（捷運站）？
- Google 評價有沒有轉化為「特色短評」而非只列數字？

### 伊歐的黃金三角檢查

- 文章有沒有「知識科普→店家推薦→FAQ」的三段式結構？
- 知識科普是否在推薦之前？
- FAQ 是否有 3-5 題？

## 七、撰寫禁令

| 禁令 | 說明 |
|------|------|
| **嚴禁 Emoji** | 對話 text 欄位中零 Emoji，用（）動作描寫替代 |
| **嚴禁機械式重複** | 關鍵字佈局要自然融入，不准同一句重複 |
| **嚴禁 AI 罐頭文** | 「本文將介紹」「綜上所述」等死板開頭一律禁止 |
| **嚴禁直貼 SOP** | 所有 SOP 規範必須包裹在角色的小說敘事動作中 |
| **使用繁體中文** | 台灣用語，「部落格」非「博客」，「搜尋」非「搜索」 |
| **全形括號** | 動作描寫用（），不用半形 () |

## 八、World 1 關卡對應表

| 關卡 | 標題 | SOP 核心概念 | 主角色 | 救援情境 |
|------|------|-------------|--------|---------|
| **1-1** | 黃金三角與關鍵字 | 黃金三角結構 + 自然關鍵字 | 伊歐（大局觀）| 斷裂的航道（缺科普前導）|
| **1-2** | 生活化標題與問句小標 | 問句式標題 + 去機器化 | 哈皮（溫度）| 冰冷的樂譜（標題太死板）|
| **1-3** | 表格化與結構設計 | 比較表格 + H2/H3 結構 | BLUE（數據）| 混亂的星圖（店家資訊雜亂）|
| **1-4** | FAQ 與收尾策略 | 結語 FAQ + 行動呼籲 | 伊歐（回航）| 無聲的燈塔（缺 FAQ）|
| **1-5** | SOP 全整合實戰 | 所有技巧綜合運用 | 三人協作 | 最終航行（完整文案）|

### 各關 SOP 重點分配

**1-1 黃金三角與關鍵字**
- 教什麼：黃金三角概念 + 關鍵字自然融入
- 伊歐：講述起航→中繼→回航的文章結構觀
- BLUE：展示關鍵字搜尋量數據
- 哈皮：示範如何讓關鍵字「唱起來」而不是「堆起來」
- **伊歐名字彩蛋**：在此關暗示 S+EO = SEO

**1-2 生活化標題與問句小標**
- 教什麼：問句式標題、生活化小標、去 AI 味
- 哈皮：主角，示範將「XX 推薦指南」改為「第一次買 XX 怎麼選？」
- BLUE：用搜尋意圖數據佐證問句標題更有效
- 伊歐：點評標題在整體航道中的定位

**1-3 表格化與結構設計**
- 教什麼：比較表格製作、H2/H3 結構、Google 評價轉化
- BLUE：主角，示範將雜亂店家資訊整理成表格
- BLUE 特技：把「4.8★」轉化為「環境安靜適合辦公，甜點精緻度高」
- 伊歐：監督結構邏輯
- 哈皮：確保表格文字有溫度

**1-4 FAQ 與收尾策略**
- 教什麼：3-5 題 FAQ 撰寫、行動呼籲、文末連結
- 伊歐：主角，講述「回航」的重要性
- BLUE：展示 FAQ 對 SEO 排名的影響數據
- 哈皮：示範 FAQ 的語氣要像「朋友在回答問題」

**1-5 SOP 全整合實戰**
- 教什麼：完整文案流程（科普→推薦→FAQ + 標題 + 表格 + 連結）
- 三人協作：各自負責自己的領域
- 最終評分：檢查所有 SOP 要素
- Practice：完整救援任務，從零建構一篇 SOP 合規文章
